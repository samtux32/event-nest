# Growth Skills

## Growth Loops

### Loop 1: Vendor Reputation Loop
```
Vendor joins → completes profile → gets bookings → earns reviews → ranks higher in search → gets more bookings
```
**Lever**: Help vendors complete their profiles. Higher completion = better search ranking = more inquiries.

### Loop 2: Customer Referral Loop
```
Customer books → has great event → leaves review → refers friends → friends book
```
**Lever**: Post-event review request (automated) → referral prompt in follow-up email.

### Loop 3: Content/SEO Loop
```
Publish planning guide → ranks in Google → customer discovers Event Nest → browses vendors → books
```
**Lever**: Scale programmatic pages (category + city combinations) and expand inspiration articles.

### Loop 4: AI Planner Loop
```
Customer uses AI planner → gets vendor recommendations → browses marketplace → books vendors → Event Nest earns fee
```
**Lever**: Promote the AI planner as the entry point for event planning. Every plan funnels to the marketplace.

### Loop 5: Vendor Referral Loop
```
Vendor joins → gets bookings → earns revenue → refers other vendors → more supply → more customer choice → more bookings
```
**Lever**: Referral programme with referral codes. Remind vendors to refer after their first confirmed booking.

## Key Metrics

### Marketplace Health
| Metric | What It Measures | Target |
|--------|-----------------|--------|
| Vendor-to-customer ratio | Supply/demand balance per city and category | 1:5–1:10 |
| Vendor response rate | % of inquiries vendors respond to | >80% |
| Vendor response time | Average time to first reply | <24 hours |
| Profile completion rate | % of vendors with complete profiles | >70% |

### Customer Funnel
| Metric | What It Measures |
|--------|-----------------|
| Browse-to-inquiry rate | % of marketplace visitors who send an inquiry |
| Inquiry-to-booking rate | % of inquiries that become confirmed bookings |
| Quote acceptance rate | % of quotes accepted by customers |
| AI planner-to-booking rate | % of plan users who book a vendor |
| Repeat booking rate | % of customers who book again for a new event |

### Vendor Funnel
| Metric | What It Measures |
|--------|-----------------|
| Signup-to-approval rate | % of vendor signups that complete profile and get approved |
| Time to first inquiry | Days from approval to first booking inquiry |
| Monthly active vendors | Vendors who log in and respond to messages |
| Vendor churn rate | % of vendors who stop using the platform per month |
| Revenue per vendor per month | Average platform fee earned per active vendor |

### Revenue
| Metric | What It Measures |
|--------|-----------------|
| GMV (Gross Merchandise Value) | Total booking value on the platform |
| Take rate | Platform fee as % of GMV (target: 12%) |
| Revenue | GMV × take rate |
| Average booking value | Average confirmed booking amount |

## Conversion Optimisation Priorities

### 1. Vendor Profile → Booking Inquiry (Highest Impact)
- Social proof: review count and average rating prominently displayed
- Clear packages with transparent pricing (not "contact for quote")
- Portfolio images that show real events
- Response time badge ("Usually responds within 2 hours")
- Availability indicator ("Available for your date")

### 2. Homepage → Marketplace Browsing
- Hero section with clear value prop and category links
- Popular categories with vendor counts
- Trust signals: total reviews, total vendors, total events planned
- AI planner CTA as secondary entry point

### 3. Vendor Signup → Completed Registration
- Show the value: "Get bookings, not leads"
- Social proof: "[X] vendors already on Event Nest"
- Simple form: email, password, business name — profile details come later
- Progress indicator through onboarding steps

### 4. AI Planner → Vendor Booking
- Plan output should link directly to relevant marketplace categories
- "Book this type of vendor" CTA next to each budget category
- Save plan functionality keeps customers engaged even if they don't book immediately

## Vendor Retention Tactics

### Early Warning Signals
- Vendor hasn't logged in for 14+ days
- Vendor has zero inquiries after 30 days
- Vendor profile is incomplete (<50% completion)
- Vendor response rate drops below 50%

### Retention Actions
| Signal | Action |
|--------|--------|
| No login (14 days) | "New customers are browsing [category] in [city]" email |
| No inquiries (30 days) | Profile optimisation tips email + offer to help |
| Incomplete profile | "Complete your profile to appear in search" reminder |
| Low response rate | "You're missing inquiries — respond to stay visible" |
| Seasonal slowdown | "[Season] is coming — update your portfolio and packages" |

### Vendor Success Milestones
Celebrate vendor achievements to keep them engaged:
- First inquiry received
- First booking confirmed
- First review received
- 5-star average rating
- 10 completed bookings
- 1 year on Event Nest

## Referral Programme

### Current System (Built)
- Vendor referral codes for vendor-to-vendor referrals
- QR codes linking to vendor profiles
- Referral tracking in vendor dashboard

### Expansion Opportunities
- **Customer referrals**: "Refer a friend planning an event" → both get a benefit
- **Vendor-to-customer**: Vendors share a link to Event Nest → their profile is featured
- **Affiliate programme**: Content creators earn commission on bookings
- **Venue referrals**: Venues refer clients to Event Nest for vendor booking
